# Удалены моки и симуляции

## Что было изменено:

### 1. Market Analyzer Service
**Файл:** `backend/src/common/scheduler/market-analyzer.service.ts`

**Было:**
- Hardcoded символы: `['BTCUSDT', 'ETHUSDT', 'BNBUSDT']`

**Стало:**
- Символы берутся из базы данных (таблица `candles`) - реальные данные
- Или из переменной окружения `ANALYZE_SYMBOLS`
- Если в БД нет данных, используется fallback на дефолтные символы

### 2. Binance Service - Historical Klines
**Файл:** `backend/src/exchanges/binance.service.ts`

**Было:**
- `getHistoricalKlines()` возвращал пустой массив `[]`

**Стало:**
- Берет реальные исторические данные из базы данных (таблица `candles`)
- Возвращает последние N свечей для символа и таймфрейма

## Результат:

✅ Все данные теперь берутся из реальных источников:
- База данных PostgreSQL
- WebSocket соединения с биржами
- API запросы

✅ Нет моков, симуляций или тестовых данных

✅ Приложение работает только с реальными данными

## Настройка:

Если нужно указать конкретные символы для анализа, добавьте в `.env`:
```
ANALYZE_SYMBOLS=BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT
```

Если переменная не указана, система автоматически возьмет символы из базы данных (из реальных свечей).
